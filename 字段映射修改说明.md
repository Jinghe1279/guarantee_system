# 后端字段映射修改说明

## 修改内容

### 1. 创建字段映射函数

在 `app.py` 中新增了 `map_database_fields_to_model_fields()` 函数，用于将前端发送的数据库字段名（下划线格式）转换为后端模型期望的字段名（英文字段名）。

### 2. 主要功能

#### 字段映射
- **项目信息**：`project_enterpriseid` → `project_number`
- **贷款信息**：`loan_apply_amount` → `application_amount`，`loan_apply_term` → `application_period`
- **财务信息**：`analysis_fin_total_assets` → `total_assets`，`analysis_fin_net_assets` → `net_assets`
- **资产负债表**：`bs_cash` → `cash_at_meeting`，`bs_ar` → `receivables_at_meeting`，`bs_inventory` → `inventory_at_meeting`
- **实控人信息**：`controller_service_years` → `industry_experience`，`controller_education` → `education_level`
- 等等...

#### 特殊处理
1. **布尔值转换**：将字符串（"是"/"否"）转换为 0/1
   - `business_is_waimao` → `is_foreign_trade` (0/1)
   - `business_is_jinshen` → `is_cautious_industry` (0/1)
   - `controller_gender` → 0/1 (0=女，1=男)
   - `controller_marital_status` → 0/1 (0=未婚，1=已婚)
   - `residence_type` → 0/1 (0=自购，1=租赁)

2. **计算字段**：
   - **核心资产**：`core_assets = bs_fixed_assets + bs_cash`
   - **用电量**：从 `electricity_items_json` 计算总用电量
   - **月还款额**：从 `existing_loans_json` 计算总月还款额
   - **日均余额**：从 `daily_avg_balance_json` 提取年均余额
   - **月净收益**：从 `is_table_annual_net_income` 计算（年收入/12）

3. **默认值处理**：为所有模型必需的字段设置默认值（0），确保即使字段缺失也不会导致错误

### 3. 修改的路由

#### `/demo` 路由
- 在接收前端数据后，先调用 `map_database_fields_to_model_fields()` 进行字段映射
- 将映射后的数据传递给 `robot.qiye_data()` 和 `robot.pinggu_qiye()` 方法

### 4. 模型必需的字段

以下字段是模型预测必需的，映射函数确保这些字段都有值：

1. `application_amount` - 申请金额
2. `net_assets` - 净资产
3. `total_assets` - 总资产
4. `core_assets` - 核心资产（固定资产+货币资金）
5. `inventory_at_meeting` - 上会时点存货
6. `industry_experience` - 从业年限
7. `local_residence_years` - 本地居住时间
8. `receivables_at_meeting` - 上会时点应收账款
9. `annual_sales` - 年销售收入
10. `electricity_consumption` - 用电量
11. `total_liabilities` - 总负债
12. `repayment_method` - 还款方案（月还本息）

## 使用说明

1. **前端发送数据**：前端继续发送数据库原始记录（下划线字段名）
2. **后端自动转换**：后端自动将字段名转换为模型期望的格式
3. **无需修改前端**：前端代码无需任何修改

## 注意事项

1. 如果数据库字段缺失，映射函数会使用默认值（0）
2. JSON字段（如 `electricity_items_json`）会自动解析并计算
3. 字符串类型的布尔值会自动转换为 0/1
4. 所有数值字段都会转换为浮点数类型

## 后续优化建议

如果发现某些字段映射不正确，可以在 `map_database_fields_to_model_fields()` 函数中：
1. 添加新的字段映射规则
2. 调整特殊处理逻辑
3. 修改默认值设置
